# UX スモークテストチェックリスト（5分版）

初めてサイトを開く人向け。**学習アプリとして「ちゃんと勉強できるか」ではなく、Phase 1 の骨格が迷わず使えるか**を確認します。

所要時間の目安: **約 5 分**  
対象 URL: https://nkhippo.github.io/English-Vocab-Chunk-Trainer/

---

## このアプリを初めて見る人へ（30秒）

このサイトは **英単語・チャンクの学習用 PWA** です。  
いまは **Phase 1** で、次の2種類の体験が中心です。

| 誰向け | 画面 | いまできること |
|---|---|---|
| **学習者（将来）** | 単語帳・学習 | 単語帳は件数確認のみ。学習モードは準備中 |
| **データ作成者（いま）** | 検証 | AI が作った候補語を、採用/却下で仕分ける |

テストでは **「壊れていないか」「説明と画面が一致しているか」** を見ます。

---

## 準備（30秒）

- [ ] 上記 URL を **シークレットウィンドウ** で開く（初回体験に近づける）
- [ ] スマホなら同じ URL を Safari / Chrome で開く
- [ ] メモ用: 各項目を ✅ / ❌ / △（わかりにくい）で記録

## 1. 初回の第一印象（1分）— ホーム

**やること**: ページを開いて、何のアプリか一目でわかるか見る。

| # | 操作 | 期待する UX | 結果 |
|---|---|---|---|
| 1-1 | ページを開く | タイトル「Vocab & Chunk Trainer」と短い説明が見える | ☐ |
| 1-2 | 初回ガイド（モーダル） | 自動で開くか、スキップ/次へ/完了で閉じられる | ☐ |
| 1-3 | 左（またはメニュー）のナビ | 「ホーム」「学習」「単語帳」「検証」「設定」に移動できる | ☐ |
| 1-4 | ホームのボタン | 「CEFR 単語帳を見る」「検証 UI を開く」が押せ、遷移する | ☐ |

**ここで失敗しやすい点**: 真っ白画面、リンクが全部 404、ガイドが閉じられない。

## 2. 単語帳（1分）— データが入っているか

**やること**: 学習データがアプリに読み込まれているか、件数で確認する。

| # | 操作 | 期待する UX | 結果 |
|---|---|---|---|
| 2-1 | 「単語帳」を開く | 見出し「CEFR 単語帳」と説明文が表示される | ☐ |
| 2-2 | レベルボタン（A1〜C1） | 各ボタンに **数字（件数）** が付いている | ☐ |
| 2-3 | **A2** を選ぶ | 「A2: ○ 件」と表示（2026-07-09 時点で **8 件前後** が目安） | ☐ |
| 2-4 | 検索欄 | グレーアウトで「検索は Phase 2 で実装します」等と分かる | ☐ |

**ここで失敗しやすい点**: 全レベル 0 件のまま（データ未投入）、件数がずっと「…」のまま。

**テストしないこと**: 単語の一覧表示・検索・詳細（未実装のため ❌ ではなく「対象外」）。

---

## 3. 検証 UI（2分）— いちばん実用的な画面

**やること**: サンプルデータを読み込み、採用/却下の流れが直感的か確かめる。  
※ 一般ユーザー向けの「勉強」機能ではなく、**候補データの仕分け用**です。

| # | 操作 | 期待する UX | 結果 |
|---|---|---|---|
| 3-1 | 「検証」を開く | 説明と「サンプルを読み込む」ボタンが見える | ☐ |
| 3-2 | **サンプルを読み込む** | 1件目の英語フレーズ・日本語訳・CEFR がカード表示される | ☐ |
| 3-3 | **Y · 採用** を押す | 次の項目に進む（または進捗が増える） | ☐ |
| 3-4 | **N · 却下** を押す | 却下として記録され、進捗が増える | ☐ |
| 3-5 | 進捗表示 | 「1 / 3（33%）」のような表示が更新される | ☐ |
| 3-6 | ショートカットの説明 | 画面下部に Y/N/E/矢印の説明がある（押さなくてよい） | ☐ |

**ここで失敗しやすい点**: ボタンを押しても反応しない、カードが真っ白、エラー文言だけ出る。

**テストしないこと**: JSON ファイルを自分で用意してアップロード（任意・上級者向け）。

## 4. 設定と言語（1分）

**やること**: 表示言語とヘルプの再表示ができるか。

| # | 操作 | 期待する UX | 結果 |
|---|---|---|---|
| 4-1 | 「設定」を開く | 表示言語の切替 UI がある | ☐ |
| 4-2 | **English** に切替 | ナビや見出しが英語になる | ☐ |
| 4-3 | **日本語** に戻す | 日本語表示に戻る | ☐ |
| 4-4 | **ガイドを再度表示** | 初回と同じガイドモーダルが開く | ☐ |

**ここで失敗しやすい点**: 言語を変えても一部だけ日本語のまま、ガイドが開かない。

---

## 5. 学習モード（30秒）— 「準備中」が正直に伝わるか

**やること**: 未実装であることがユーザーに誤解なく伝わるか（ネガティブテスト）。

| # | 操作 | 期待する UX | 結果 |
|---|---|---|---|
| 5-1 | 「学習」を開く | 「Phase 2」「準備中」等、**まだ学習できない**ことが明示される | ☐ |
| 5-2 | 期待値の確認 | フラッシュカードや問題が出てこない（出たらバグ） | ☐ |

**ここで失敗しやすい点**: 何も説明なく空画面だけ、または「学習できる」と誤解する文言。

## 合否の目安（5分終了時）

| 判定 | 条件 |
|---|---|
| **合格** | 2（単語帳件数）・3（検証サンプル）・4（言語切替）がすべて ✅ |
| **要修正** | 上記のいずれかが ❌、または初回ガイド/ナビが使えない |
| **対象外** | 学習モードの中身、単語検索、SRS、AI 添削 — 未実装のため不合格扱いにしない |
